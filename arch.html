Reconciliation Orchestration Engine

Internal Components, APIs and Integrations

High-level view of how the Reconciliation Orchestration Engine receives API calls, builds execution plans, coordinates batches, and integrates with schema inference, extraction, matching, comparison, and results storage.

REST API Layer

▼ API calls (start, cancel, status)
Reconciliation Orchestration Engine
1. Northbound Interface
Job Submission Facade
Entry point for Jobs API calls. Validates inputs (rule_set_id, dataset ids, mappings) against metadata. Delegates to Run Context Builder.
Run Context Builder
Hydrates RunContext with resolved systems, datasets, mappings, rule set and execution options (batch size, parallelism, timeouts).
Run Registry
Creates ReconciliationRun records, assigns run_id, sets initial status (PENDING), and persists to metadata DB.
2. Core Orchestration Services
Plan Builder
Derives ExecutionPlan from RuleSet, Mappings, and Datasets. Defines ordered steps per dataset pair: schema inference, extraction, matching, comparison, persist results.
Execution Controller
Main state machine for a run. Drives plan steps and transitions: PENDING → RUNNING → COMPLETED / FAILED / CANCELLED. Coordinates batches and step lifecycle.
Progress Tracker
Tracks counts for total, processed, matched, unmatched and discrepancies. Exposes snapshots for GET /jobs/{id}.
Error & Retry Manager
Classifies errors (transient vs fatal), applies retry policies (max attempts, backoff), decides when to abort or continue with warnings.
3. Execution & Coordination Layer
Batch Orchestrator
Splits work into BatchContext units (e.g. 10k records). For each dataset pair, coordinates source/target extraction and routes record streams to matching.
Task Dispatcher
Dispatches batch tasks to workers (in-process async/thread pool; pluggable queue in future). Enforces concurrency limits and tracks active tasks.
SchemaInferenceAdapter
Integration wrapper around OracleSchemaInferrer and MongoDBSchemaInferrer. Executes schema steps referenced in ExecutionPlan.
ExtractionAdapter
Wraps OracleDataExtractor and MongoDBDataExtractor. Applies filters, paging and transformations; returns iterators/streams to Batch Orchestrator.
MatchingAdapter
Wraps MatchingEngine.match_records(...). Uses configured matching keys to produce matched/unmatched sets for each batch.
ComparisonAdapter
Uses FieldMappingService and comparators to compare fields for matched pairs. Routes discrepancies and stats to ResultsRepository.
Shared Internal Models
RunContext
Per-run context: run_id, job_id, rule_set, mappings, datasets, execution options. Passed across engine components.
ExecutionPlan
Step graph for reconciliation: ordered and dependent steps per dataset pair. Drives Execution Controller behaviour.
BatchContext
Per-batch metadata: batch_id, dataset pair, paging/window bounds. Used by Batch Orchestrator and Task Dispatcher.
